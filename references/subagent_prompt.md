# 文本块知识提取 — Subagent 任务模板（两阶段）

## 两阶段设计

对每个文本块执行两次 Agent 调用：

**Phase 1 — 关键词提取**：只读文本，识别知识型条目的关键词，不调用 WebSearch。
**Phase 2 — 检索撰写**：仅对脚本过滤后的新关键词做 WebSearch 并生成 YAML。

---

## Phase 1：关键词提取

### 任务目标

从文本块中识别**知识型条目**的关键词，保存为纯文本列表（每行一个关键词）。

### 核心判断标准

**"一个不熟悉背景的读者，看到这个词会需要查资料才能理解吗？"**

- 需要查资料 → 是知识点，应当提取（如"扎染"、"傩戏"、"土楼"）
- 不需要 → 不是知识点，不提取（如主角名"翠翠"）

### 输入

- 文本块文件路径: `{chunk_filepath}`
- 书名: `{book_name}`
- 输出路径（关键词文件）: `{keywords_filepath}`

### 输出格式

纯文本文件，每行一个关键词：

```
扎染
傩戏
土楼
油纸伞
```

### 分类筛选

- 物件·工艺 — 具有民族/地方特色的物品、工具、手工艺品等（排除普通日用品）
- 习俗·仪式 — 有特定文化背景的风俗、节日、祭祀、禁忌
- 概念·观念 — 特有的思想、信仰、价值观、民间说法
- 事件·典故 — 具有历史或文化意义的事件、传说、典故
- 人物 — 仅限于真实历史人物或文化符号人物（排除书中叙事角色）
- 地点 — 仅限于具有特定文化、历史、地理意义的地点

### 明确排除

- 书中普通人物名字（除非是真实历史人物）
- 普通日常物品、通用动作或状态
- 仅作为叙事背景出现、无需额外解释的概念

### 重要原则

1. **禁止使用 WebSearch** — 本阶段仅做识别，不做检索
2. **按实提取** — 不设上下限，文本块中有多少就提取多少
3. 不设数量上下限，实报实销

---

## Phase 2：WebSearch + YAML 输出

### 任务目标

对指定的关键词列表进行互联网检索，结合书中上下文撰写解释，输出标准 YAML。

### 输入

- 文本块文件路径: `{chunk_filepath}`
- 书名: `{book_name}`
- 需要处理的关键词列表: `{new_keywords}`
- 输出路径: `{output_yaml_path}`

### 输出格式 — 严格遵循

保存为 **YAML** 文件（扩展名 `.yaml`），每条记录包含 4 个字段：

| 字段名 | 说明 |
|--------|------|
| `名词` | 知识条目的中文名称 |
| `解释` | 综合解释（50-300 字） |
| `书中原文` | 书中出现的完整句子 |
| `网络来源` | 检索到的 URL（每行一个） |

### 禁止的格式

- ❌ 使用 `term:` 代替 `名词:`
- ❌ 在顶层包裹 `entries:`、`source:`、`nouns:` 等父键
- ❌ 添加 `type:`、`aliases:`、`category:` 等额外字段
- ❌ 字段名使用英文

### 正确示例

```yaml
- 名词: 扎染
  解释: |
    中国传统的手工染色技术之一，以绞扎、浸染为主要工艺。
    扎染的图案自然晕染，每件作品独一无二。
  书中原文: |
    她穿着一件扎染的蓝布衣裳，在人群中显得格外与众不同，那衣裳上的花纹像是天上的云彩。
  网络来源: |
    https://baike.baidu.com/item/扎染
    https://zh.wikipedia.org/wiki/扎染

- 名词: 傩戏
  解释: |
    中国古老的戏剧形式，表演者戴面具……
  书中原文: |
    正月里，村里请了傩戏班子来唱了三天三夜，锣鼓喧天，震得人耳朵嗡嗡响。
  网络来源: |
    https://baike.baidu.com/item/傩戏
```

### YAML 书写规则

- 每条记录以 `- 名词:` 开头
- 每个字段缩进 2 空格
- 短文本可以跟在 `: ` 后面直接写
- 多行文本用 `: |` 换行，内容缩进 4 空格
- 字段顺序固定：名词 → 解释 → 书中原文 → 网络来源

### 工作步骤

1. 使用 Read 工具阅读文本块
2. 对每个关键词使用 WebSearch 进行互联网检索（每次 1-2 个，避免过于频繁）
3. 综合书中内容 + 检索结果撰写解释
4. 使用 Write 工具输出 YAML 文件

### 重要原则

1. **只处理收到的关键词列表**，不要自行添加新关键词
2. **每个关键词都需要 WebSearch**，除非明确说明已在列表中排除的
3. 如果网络检索不到，在"解释"中注明"未检索到网络资料"
4. 禁止自行编造信息
5. 按实提取，有多少写多少
