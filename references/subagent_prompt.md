# 文本块知识提取 — Subagent 任务模板

这是一个 subagent 的任务说明。主 skill 会在处理每个文本块时，以此模板为基础创建 subagent 的 prompt。

## 任务目标

从给定的电子书文本块中提取**知识型条目**——即具有文化、历史、民族、地方特色的知识点，而非普通的叙事名词。检索互联网获取相关概念解释，结合书中上下文进行总结。

### 核心判断标准：这是个"知识点"还是"普通名词"？

问自己一个问题：**"一个不熟悉这些背景的读者，看到这个词会需要查资料才能理解吗？"**

- 需要查资料 → 是知识点，应当提取（如"扎染"、"傩戏"、"土楼"）
- 不需要，仅作为叙事要素出现 → 不是知识点，不应提取（如主角名"翠翠"、"张三"）

## 输出格式

保存为一个 **YAML** 文件（扩展名 `.yaml`，不要用 `.json`）。YAML 不需要操心引号转义，多行文本用 `|` 符号直接书写。

```yaml
- 名词: 扎染
  解释: |
    中国传统的手工染色技术之一，以绞扎、浸染为主要工艺。
    扎染的图案自然晕染，每件作品独一无二。
  书中原文: |
    她穿着一件扎染的蓝布衣裳，在人群中显得格外与众不同，那衣裳上的花纹像是天上的云彩。
  网络来源: |
    https://baike.baidu.com/item/扎染
    https://zh.wikipedia.org/wiki/扎染

- 名词: 傩戏
  解释: |
    中国古老的戏剧形式，表演者戴面具……
  书中原文: |
    正月里，村里请了傩戏班子来唱了三天三夜，锣鼓喧天，震得人耳朵嗡嗡响。
  网络来源: |
    https://baike.baidu.com/item/傩戏
```

**YAML 书写规则（很简单）：**
- 每条记录以 `-` 开头，顶格写
- 字段名（名词/解释/书中原文/网络来源）缩进 2 空格，后面跟 `: ` 再加内容
- 普通短文本可以跟在 `: ` 后面直接写
- **多行文本（解释、书中原文、网络来源）** 用 `: |` 后面换行，内容缩进 4 空格，可以任意写，完全不用操心引号
- 空的字段留空或写 `: ""` 即可
- 字段顺序保持：名词 → 解释 → 书中原文 → 网络来源

## 工作步骤

### 步骤 1：阅读文本块

使用 Read 工具阅读文本块文件，全面理解内容。

### 步骤 2：识别知识型条目

以"是否值得查资料理解"作为筛选标准，从文本中识别以下类别的知识点：

- **物件·工艺** — 具有民族特色、地方特色的物品、工具、手工艺品、乐器、服饰、食物等（如银饰、蜡染、油纸伞）。**注意**：普通日用品不算（如"碗"、"筷子"），除非有特殊文化含义
- **习俗·仪式** — 有特定文化背景的风俗、传统、节日、礼仪、祭祀、禁忌（如"跳月"、"成年礼"、"祭山神"）
- **概念·观念** — 特有的思想、哲学、信仰、价值观、民间说法（如"天地君亲师"、"万物有灵"）
- **事件·典故** — 具有历史或文化意义的事件、传说、典故（如"走西口"、"赶秋"）
- **人物** — 仅限于**真实历史人物**或**具有显著文化符号意义的虚构人物**（如"屈原"、"阿凡提"）。**排除**：仅作为书中叙事角色的普通主人公、配角名字
- **地点** — 仅限于具有特定文化、历史、地理意义的地点（如"茶马古道"、"吊脚楼"）。**排除**：普通故事发生地、虚构场景名

### 明确排除清单

以下类型的名词**不应**提取：
- 书中普通人物名字（无论主角配角），除非是真实历史人物
- 普通日常物品（桌子、水、房子等）
- 通用动作或状态（走路、吃饭、高兴等）
- 仅作为叙事背景出现、无需额外解释的概念

### 选择优先级

1. 高优先级：具有民族/地方特色，不查阅资料就无法理解的条目
2. 中优先级：具有一定文化含义，但可能部分读者已知的条目
3. 低优先级：书中重点描绘的特定知识（如某种传统工艺的详细工序）

不设数量上下限，文本块中实际有多少知识型条目就提取多少个，实报实销。

### 步骤 3：互联网检索

对于每个识别出的名词，使用 WebSearch 工具检索其概念和背景信息。

检索策略：
- 每次检索 1-2 个名词，避免过于频繁的并发请求
- 使用"名词 + 书籍背景"的检索词，以获得更精准的结果
- 如果首次检索结果不理想，更换检索词再试一次

### 步骤 4：撰写解释并输出

对于每个名词，综合书中原文和网络检索结果撰写解释：

- **解释** 应该简洁明了（50-300 字），说明该名词是什么、在书中的角色/意义
- **书中原文** 必须是完整的原文句子，让人能看懂上下文来源，不能是截断片段
- **网络来源** 列出检索到的相关 URL（权威来源优先，如百科、学术网站）

### 重要原则

1. **禁止自行编造信息** — 如果网络检索不到某个条目，只需在"解释"中注明"未检索到相关网络资料"，并仅基于书中内容进行解释
2. **按实提取** — 不设数量上下限，文本块中有多少知识型条目就提取多少，实报实销。主角名、日常物品、通用概念一律不收
3. **文化敏感性** — 对于涉及民族文化、宗教习俗的条目，保持客观描述，避免主观评判
4. **语言匹配** — 如果书中条目是外文，保留原文并标注中文翻译
5. **JSON 引号陷阱** — `"书中原文"`、`"解释"` 等字段内容中如果有双引号（如书中对话"你好的"含义），务必转义为 `\"`，否则 JSON 会断裂。输出前严格校验引号配对
